Over de Machtswet

Een wiskundig kader om het langetermijnprijsgedrag van Bitcoin te begrijpen door de lens van netwerkadoptie.

Wat is een Machtswet?

Een machtswet is een wiskundige relatie waarbij de ene grootheid varieert als een macht van de andere:

y = A × xβ

Machtswetten komen overal in de natuur en menselijke systemen voor:

  • Stadsgrootten volgen een machtswetdistributie (Wet van Zipf)
  • Aardbevingsmagnitudes volgen de Gutenberg-Richter-wet
  • Woordfrequenties in talen volgen machtswetdistributies
  • Netwerkeffecten vertonen vaak machtswetgroei (Wet van Metcalfe)

Op een log-log schaal verschijnen machtswetrelaties als rechte lijnen. Dit is precies wat we waarnemen bij de prijs van Bitcoin door de tijd heen.

Het Machtswetgedrag van Bitcoin

Sinds het ontstaan heeft de prijs van Bitcoin een opmerkelijk consistente machtswetrelatie met tijd gevolgd. Wanneer log(prijs) wordt uitgezet tegen log(tijd sinds genesis), vormen de gegevens een bijna rechte lijn met een R-kwadraat van ongeveer 95%.

Waarom zou dit voorkomen?

De machtswetrelatie ontstaat waarschijnlijk door de netwerkeffecten van Bitcoin. Naarmate meer mensen Bitcoin adopteren, wordt het netwerk waardevoller (vergelijkbaar met de Wet van Metcalfe, waarbij de netwerkwaarde schaalt met het kwadraat van het aantal gebruikers). Het consistente groeitempo suggereert dat adoptie voorspelbare patronen volgt over lange tijdschalen.

Het Model

Santostasi/Perrenod Machtswetmodel

Bron: Giovanni Santostasi's onderzoek naar de Bitcoin Machtswet Theorie. Parameters van bitcoinpower.law

trendprijs = 10-16,493 × dagen5,688

Dit model gebruikt dagen sinds genesis (3 januari 2009) als tijdsvariabele. Belangrijkste kenmerken:

  • β = 5,688 (exponent)
  • log10(A) = -16,493 (snijpunt)
  • R² ≈ 95%
  • Huidige trendwaarde ~$130K (februari 2026)

Terugkeer naar het Gemiddelde

Een van de belangrijkste aspecten van het machtswetmodel is het begrijpen van terugkeer naar het gemiddelde. De prijs van Bitcoin volgt de trendlijn niet perfect; hij oscilleert eromheen, soms dramatisch.

De Elastiek-analogie

Stel je voor dat de prijs met een elastiek aan de trendlijn is bevestigd. Het kan ver in beide richtingen uitrekken tijdens bubbels of crashes, maar hoe verder het uitrekt, hoe sterker de trekkracht terug naar de trend. Dit voorspelt geen kortetermijnbewegingen, maar historisch gezien zijn extreme afwijkingen altijd gecorrigeerd.

Methodologie

Databronnen

  • Live prijs: CoinGecko API

Berekening Standaardafwijking

De σ (sigma) banden worden dynamisch berekend op basis van de historische gegevens:

  1. Bereken voor elk historisch datapunt log10(werkelijke_prijs) - log10(trendprijs)
  2. Deze "log-residuen" tonen hoe ver de prijs afweek van de trend
  3. Bereken de standaardafwijking van deze residuen
  4. Gebruik deze σ om banden te maken: trend × 10±kσ

Genesis Blok

Beide modellen gebruiken 3 januari 2009, 00:00:00 UTC als de genesis-tijdstempel (blok 0).

Beperkingen & Kanttekeningen

Dit is geen financieel advies. Het machtswetmodel is een empirische observatie, geen garantie voor toekomstige prestaties.

  • Empirisch, niet fundamenteel: De machtswet is een curve-fit op historische gegevens. Het is niet afgeleid van eerste beginselen.
  • Extrapolatierisico: Patronen uit het verleden hoeven niet oneindig door te gaan. Het model kan falen.
  • Volatiliteit: Zelfs als de trend standhoudt, kunnen werkelijke prijzen 50-80% in beide richtingen afwijken.
  • Externe factoren: Veranderingen in regelgeving, technologische disruptie of macro-economische gebeurtenissen kunnen het traject van Bitcoin veranderen.
  • Overlevingsbias: We analyseren Bitcoin omdat het heeft overleefd. Mislukte cryptovaluta's verschijnen niet in deze analyse.
  • Modelonzekerheid: Verschillende parameterkeuzes (beta, genesis-tijd, databronnen) leveren verschillende projecties op.

Technische Referentie

// Genesis blok
const GENESIS = new Date('2009-01-03T00:00:00Z');

// Santostasi/Perrenod Model (van bitcoinpower.law)
const dagenSindsGenesis = (Date.now() - GENESIS.getTime()) / (1000 * 60 * 60 * 24);
const trendPrijs = Math.pow(10, -16.493) * Math.pow(dagenSindsGenesis, 5.688);

// Sigma-banden (waarbij sigma is berekend uit historische log-residuen)
const bovenBand = trendPrijs * Math.pow(10, sigma);
const onderBand = trendPrijs * Math.pow(10, -sigma);
